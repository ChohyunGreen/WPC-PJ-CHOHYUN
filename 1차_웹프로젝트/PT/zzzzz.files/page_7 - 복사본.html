인터랙티브 프로토타입

피그마(Figma)를 활용하여 실제 웹사이트와 유사하게 동작하는 인터랙티브 프로토타입을 제작했습니다. 이 프로토타입은 페이지 전환, 버튼 클릭 시의 반응, 스크롤 애니메이션 등 다양한 상호작용 요소를 포함하여 사용자 경험의 흐름을 시뮬레이션합니다.

페이지 전환

마우스 클릭으로 페이지 이동 및 히스토리 관리

클릭 반응

버튼, 링크, 입력 필드 등의 상호작용 구현

스크롤 애니메이션

부드러운 스크롤 및 요소 표시/숨김 효과

사용자 테스트

개발 전 실제 사용자 경험 테스트 및 개선

프로토타입의 가치

인터랙티브 프로토타입을 통해 개발 전 사용자 테스트를 진행하여 잠재적인 문제점을 조기에 발견하고 개선할 수 있었습니다. 예를 들어, 상품 상세 페이지에서 '옵션 선택' UI가 직관적이지 않다는 피드백을 받아 드롭다운 메뉴 대신 버튼형 선택지로 변경하여 사용성을 개선했습니다.

컴포넌트 기반 설계

피그마의 컴포넌트 기능을 적극 활용하여 디자인 시스템을 구축하고 재사용 가능한 UI 요소를 설계했습니다. 버튼, 입력 필드, 카드 UI 등 반복적으로 사용되는 요소들을 컴포넌트로 정의하고, 이를 라이브러리로 관리하여 디자인의 일관성을 유지하고 작업 효율성을 극대화했습니다.

컴포넌트의 장점

컴포넌트 기반 설계는 향후 디자인 변경이나 기능 확장에 유연하게 대응할 수 있는 기반이 됩니다. 디자인 시스템의 일관성을 유지하면서 개발 속도를 향상시키며, 팀원 간의 협업을 용이하게 합니다. 이는 특히 대규모 프로젝트에서 컴포넌트의 재사용성으로 인한 효율성 증가가显著합니다.
